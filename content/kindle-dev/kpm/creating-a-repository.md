# A note to developers
Under most circumstances, this is unecessary and it is sufficient to submit packages to the [official repository](https://github.com/KindleModding/repo)

# Creating a Repository
A KPM repository can be created using the [`kpm-helper.py`](https://github.com/KindleModding/KPM/blob/main/kpm-helper.py) script:
```sh
# Step 1. Create a folder to store your package contents
mkdir example_repository
# Step 2. Initialise the package folder
python kpm-helper.py repo init ./example_repository
```

However, it is recommended that you fork the [example repository](https://github.com/KindleModding/kpm-repository-template) instead.

## Adding a package to your repository
```sh
python kpm-helper.py repo add ./example_repository PACKAGE_PATH
```

Where the `PACKAGE_PATH` is the path to a `.kpkg` file generated by `kpm-helper.py`
